## Tokenizers: BPE, WordPiece, SentencePiece
### 
- **Word-level tokenization fails** because it requires an enormous vocabulary to include every possible word.
- **Character-level tokenization fails** because it splits every word into individual characters, creating very long sequences.

- **Subword tokenization succeeds** because it balances both approaches. It keeps common words as single tokens while breaking rare words into meaningful pieces. `unhappiness` => `['un','happi','ness']`

**Byte Pair Encoding** (BPE) is a subword tokenization algorithm. Start with individual characters. Count every adjacent pair in the training corpus. Merge the most frequent pair into a new token. Repeat until you reach your target vocabulary size.

## Scaling: Distributed Training, FSDP, DeepSpeed

**Data Parallelism** is the simplest way to use multiple GPUs. Every GPU stores a complete copy of the model, but each GPU trains on a different portion of the training data. After processing its data, each GPU computes gradients, and these gradients are averaged across all GPUs so that every copy of the model is updated in the same way. This makes training much faster because many data samples are processed simultaneously. However, since every GPU still stores the entire model, it does not reduce memory usage per GPU


**Tensor Parallelism** solves the memory problem by splitting each neural network layer across multiple GPUs. Instead of storing a complete weight matrix on one GPU, each GPU stores only a portion of it and performs part of the computation. The partial results are then combined to produce the final output. This allows very large models to fit across several GPUs. 

**Pipeline Parallelism**, the model is divided by layers instead of splitting individual layers. For example, GPU 1 processes the first few layers, GPU 2 processes the next set of layers, and so on. The output from one GPU is passed to the next like an assembly line in a factory. This greatly reduces memory usage because each GPU stores only a portion of the model.

**FSDP (Fully Sharded Data Parallel)** Instead of each GPU storing the entire model, every GPU stores only a small fraction of the model's weights, gradients, and optimizer states. Whenever a layer needs to be computed, the missing parameters are temporarily gathered from other GPUs, and after computation they are discarded again.

**Mixed Precision Training**, where the model performs most calculations using 16-bit numbers (FP16 or BF16) instead of standard 32-bit numbers (FP32). Since 16-bit numbers require only half the memory, this significantly reduces GPU memory usage and speeds up computation. However, to maintain numerical accuracy, the optimizer still stores a master copy of the weights in FP32.


Training the world's largest LLMs combines Data Parallelism, Tensor Parallelism, and Pipeline Parallelism together, a strategy called **3D Parallelism**.
